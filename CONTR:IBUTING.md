## Finding and Creating Issues

1. **Identify a feature or bug** within the repository that can be improved or fixed.
2. **Create a new GitHub issue** describing the specific problem or enhancement.
3. **Include the following notice** in your issue description:

> This issue is limited only to the creator of this issue. This means that only the issue author can attempt to solve this issue. If you would like to work on it, please create another issue with the same contents and refer to issue #743 for more information.

## Requirements

- Issues should be focused and specific
- Each issue should address only one feature or bug
- Pull requests without corresponding issues will not be considered

## Additional Notes

- Make sure to follow the issue creation format
- Include all necessary context and details for the proposed changes
